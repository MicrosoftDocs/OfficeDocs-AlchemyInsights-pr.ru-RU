1. <span data-ttu-id="19ed9-104">[Немедленный сброс пароля учетной записи](https://support.office.com/article/7a5d073b-7fae-4aa5-8f96-9ecd041aba9c)</span><span class="sxs-lookup"><span data-stu-id="19ed9-104">[Reset the user's password](https://support.office.com/article/7a5d073b-7fae-4aa5-8f96-9ecd041aba9c) immediately.</span></span> <span data-ttu-id="19ed9-105">Не сообщайте новый пароль в сообщении электронной почте конечному пользователю.</span><span class="sxs-lookup"><span data-stu-id="19ed9-105">Do not communicate the new password through email to the end user.</span></span>

3. <span data-ttu-id="19ed9-107">Удалите все подозрительные [правила для папки "Входящие"](https://support.office.com/article/1433E3A0-7FB0-4999-B536-50E05CB67FED), заданные для почтового ящика.</span><span class="sxs-lookup"><span data-stu-id="19ed9-107">Remove any suspicious [inbox rules](https://support.office.com/article/1433E3A0-7FB0-4999-B536-50E05CB67FED) set within the mailbox.</span></span>

4. <span data-ttu-id="19ed9-108">Если пользователю запрещено отправлять электронную почту, [перейдите на страницу "Пользователи с ограниченным доступом", чтобы разблокировать учетную запись](https://protection.office.com/?hash=/restrictedusers).</span><span class="sxs-lookup"><span data-stu-id="19ed9-108">If the user is blocked from sending email, [go to the Restricted Users to unblock the account](https://protection.office.com/?hash=/restrictedusers).</span></span> <span data-ttu-id="19ed9-109">После завершения процедуры отправка сообщений должна возобновиться в течение часа.</span><span class="sxs-lookup"><span data-stu-id="19ed9-109">Once done, the user should be able to resume sending messages within 1 hour.</span></span>

5. <span data-ttu-id="19ed9-110">Удалите учетную запись этого пользователя из всех [групп с административными ролями](https://support.office.com/article/eac4d046-1afd-4f1a-85fc-8219c79e1504) до тех пор, пока вы не убедитесь в том, что его учетная запись больше не скомпрометирована.</span><span class="sxs-lookup"><span data-stu-id="19ed9-110">Remove the user account from any [administrative role groups](https://support.office.com/article/eac4d046-1afd-4f1a-85fc-8219c79e1504) until you are confident that the account is no longer compromised.</span></span>
