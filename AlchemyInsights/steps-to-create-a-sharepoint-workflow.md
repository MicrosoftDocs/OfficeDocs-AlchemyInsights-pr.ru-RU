# <a name="steps-to-create-a-sharepoint-workflow"></a>Действия для создания рабочего процесса SharePoint

- Если вы не можете добавить рабочий процесс утверждения SharePoint 2010, ознакомьтесь со статьей [не удается добавить рабочий процесс утверждения по умолчанию 2010](https://docs.microsoft.com/alchemyinsights/can-t-add-default-2010-approval-workflow).
- Создание рабочих процессов:
    - [Рабочий процесс 2013 из SharePoint Designer](https://docs.microsoft.com/sharepoint/dev/general-development/creating-a-workflow-by-using-sharepoint-designer-and-the-sharepoint-wo)
    - [Рабочий процесс 2010 из SharePoint Designer](https://support.office.com/article/introduction-to-designing-and-customizing-workflows-32c9c0bf-5e20-4f74-8b9c-d3ea79f2962b)
    - [Общие сведения о рабочем процессе SharePoint](https://support.office.com/article/introduction-to-sharepoint-workflow-07982276-54e8-4e17-8699-5056eff4d9e3)

- Устранение неполадок и Отладка рабочих процессов:
    - Отладка рабочих процессов похожа на предыдущие версии.  Более подробную информацию можно узнать в статье [Отладка рабочих процессов SharePoint](https://docs.microsoft.com/sharepoint/dev/general-development/debugging-sharepoint-server-workflows).
    - Более подробную информацию об устранении неполадок можно найти в статье [Устранение ошибок проверки рабочих процессов SharePoint Server](https://docs.microsoft.com/sharepoint/dev/general-development/troubleshooting-sharepoint-server-workflow-validation-errors-in-visio).
