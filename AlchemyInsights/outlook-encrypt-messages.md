# <a name="encrypt-email-messages-in-outlook"></a>Шифрование сообщений электронной почты в Outlook

<p><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">Шифрование сообщений Office 365 основано на Microsoft Azure Rights Management (Azure RMS), который входит в Azure Information Protection. Если ваша подписка включает Azure Rights Management или Azure Information Protection, <strong style="mso-bidi-font-weight: normal;">вам не нужно выполнять какие – либо действия для включения или отключения</strong> службы управления правами вручную.</span></p> <p><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">В зависимости от отзывов клиентов мы больше не сможете включить правила для почтовых ящиков Exchange для автоматического шифрования исходящей электронной почты с определенным типом конфиденциальной информации в клиенте по умолчанию. &nbsp; Вместо этого мы предоставляем подробные инструкции по выполнению йоурселвес. &nbsp;Дополнительные сведения о том, как создать правило транспорта для шифрования конфиденциальной информации, можно найти в этой <a href="https://aka.ms/OmeEtr">статье</a>.</span><u></u><span style="text-decoration: line-through;"></span></p> <ul> <li style="text-indent: -.25in; mso-list: l0 level1 lfo1;"><span style="font-size: 10.5pt; font-family: Symbol; mso-fareast-font-family: Symbol; mso-bidi-font-family: Symbol;"><span style="mso-list: Ignore;">&nbsp;При &nbsp; Outlook в Интернете (ранее Outlook Web App): при создании сообщения электронной почты просто нажмите кнопку Защита в OWA. &nbsp; &nbsp; </span> </span> <span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;"> <strong style="mso-bidi-font-weight: normal;"></strong> <strong></strong> Этот параметр будет применяться по &lsquo;умолчанию —&rsquo; не пересылать разрешения. Нажмите кнопку <strong>изменить разрешение</strong> и выберите <strong>Шифрование</strong> только для шифрования сообщения.</span></li> <li style="text-indent: -.25in; mso-list: l0 level1 lfo1;"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">&nbsp;Если используется <strong style="mso-bidi-font-weight: normal;">клиент Outlook</strong>: для отправки зашифрованного сообщения из Outlook 2013 или 2016 или Outlook 2016 для Mac, выберите параметры &agrave; разрешения, а затем выберите необходимый вариант защиты. &nbsp; &nbsp;</span></li> <li style="text-indent: -.25in; mso-list: l0 level1 lfo1;"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">&nbsp;&nbsp; Чтобы <strong style="mso-bidi-font-weight: normal;">автоматически шифровать все сообщения</strong> , отправленные определенным получателям или партнерским организациям, необходимо создать правило транспорта для поток обработки почты в центре администрирования Exchange. Подробные инструкции приведены в этой <span style="color: black;"><a href="https://docs.microsoft.com/office365/securitycompliance/define-mail-flow-rules-to-encrypt-email#create-a-mail-flow-rule-to-encrypt-email-messages-with-the-new-ome-capabilities">статье поддержки</a></span></span></li> </ul>
