<p><span data-ttu-id="2d11a-103"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">Если вы получаете сообщение об отказе в доступе при попытке перейти в центр администрирования SharePoint или OneDrive, убедитесь, что вы <a href="https://docs.microsoft.com/en-us/office365/admin/subscriptions-and-billing/assign-licenses-to-users?view=o365-worldwide&amp;tabs=One">назначили пользователю лицензию </a>. Если у пользователя есть лицензия, убедитесь, что <a href="https://docs.microsoft.com/en-us/office365/admin/add-users/about-admin-roles?view=o365-worldwide">ей назначена роль администратора</a> , которая может получать доступ к центрам администрирования.</span></span><span class="sxs-lookup"><span data-stu-id="2d11a-103"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">If you are receiving an access denied message when attempting to browse to a Sharepoint/OneDrive Admin Center, please make sure that you <a href="https://docs.microsoft.com/en-us/office365/admin/subscriptions-and-billing/assign-licenses-to-users?view=o365-worldwide&amp;tabs=One">assign a license to the user </a>. If the user has a license, you should also make sure they are <a href="https://docs.microsoft.com/en-us/office365/admin/add-users/about-admin-roles?view=o365-worldwide">assigned an administrator role</a> that can access the admin centers.</span></span></span></p>  <p style="orphans: 2; -webkit-text-stroke-width: 0px; word-spacing: 0px;"><span data-ttu-id="2d11a-104"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">Эта проблема также может возникать, если пользователь удален и повторно создан с тем же именем участника-пользователя (UPN). If users have already signed in to SharePoint, and then are moved to a different OU and resynced with SharePoint, they may experience this problem.</span></span></span></span></p>  <ul style="orphans: 2;
